# SCD Assessment Repository

## Overview

This repository contains comprehensive coursework and assessments for Software Construction & Design (SCD). It covers multiple software engineering concepts, design patterns, and practical implementations of system design principles.

## 📁 Project Structure

The repository is organized into the following sections:

### 1. **Code Readability** 
Located in: `/Code_readability/`

Demonstrates best practices in writing clean, maintainable code:
- **bad_code.py** - Example of poorly written code with readability issues
- **good_code.py** - Refactored version showing coding standards and best practices

**Key Topics:**
- Variable naming conventions
- Code documentation
- Function decomposition
- Commenting practices

### 2. **Modular Programming**
Located in: `/moduler-programming/`

Implements a modular student management system showcasing separation of concerns:
- **main.py** - Application entry point
- **login.py** - Authentication module
- **student.py** - Student data management
- **report.py** - Reporting functionality
- **result.py** - Grade/result handling
- **modules & responsibilities.txt** - Module documentation

**Purpose:** Demonstrates how to break down a system into independent, reusable modules with clear responsibilities.

### 3. **Library Management System**
Located in: `/library-managment-system/`

A complete system design and implementation project:
- **code.py** - Main implementation of the library management system
- **Stakeholders.txt** - Stakeholder analysis and requirements

**Features:**
- User management
- Book catalog
- Lending and return operations
- Stakeholder requirements documentation

### 4. **UML & Design**
Located in: `/UML/`

Visual system design documentation:
- UML diagrams for system architecture
- Class relationships and interactions
- System component overview

### 5. **Use Case Analysis**
Located in: `/Use-Case-Analysis/`

**Use-case.txt** - Detailed use case documentation describing:
- System actors
- User interactions
- Business processes
- Functional requirements

### 6. **Version Control**
Located: `Version-control.txt`

Documentation on version control practices and Git workflow used in this assessment.
